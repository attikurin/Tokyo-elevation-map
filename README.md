# 東京 高低マップ Web アプリ集

東京都全62区市町村のインタラクティブな高低（標高）マップを目指すWebアプリ集です。ポータルページから地図クリックまたは一覧で自治体を選択できます。現在は4自治体が完成済みで、残り58自治体は順次追加予定です。

## 📂 プロジェクト構成

```
/
├── index.html              # 🗾 ポータルページ（都地図＋一覧）
├── manifest.json           # PWA マニフェスト（ポータル）
├── sw.js                   # Service Worker（共通実装）
├── icon.svg                # PWA アイコン（ポータル）
│
├── fuchu/                  # 府中市版（完成形・保全／単一HTML）
├── itabashi/               # 板橋区版（完成形・保全／単一HTML）
├── minato/                 # 港区版（完成形・保全／単一HTML）
├── koto/                   # 江東区版（完成形・保全／単一HTML）
│  （各フォルダに index.html / manifest.json / sw.js / icon.svg）
│
├── app/                    # 🆕 共通コアテンプレート（新規自治体用）
│   ├── map.html            # データ駆動型の汎用マップ
│   ├── manifest.json
│   ├── sw.js
│   └── icon.svg
│
├── data/                   # 🆕 自治体別データ
│   ├── tokyo-index.json    # 全62自治体のメタデータ
│   ├── chiyoda.json        # 千代田区（共通コア方式サンプル）
│   └── README.md           # 新規自治体追加ガイド
│
├── offline/index.html      # 🔌 オフライン版の詳細説明ページ
└── README.md
```

各自治体ページは**単一HTMLで完結**（CSS・JS・データ内蔵）しており、個別に使用可能です。

## ✅ 完了機能（全バージョン共通）

### 4つのコア機能を1ページに統合
1. **2D 標高マップ** – 国土地理院の色別標高図 + 陰影起伏図
2. **3D 立体表示** – MapLibre GL JS + カスタムプロトコル（GSI DEM PNG → terrain-rgb 変換）
3. **断面図** – 地図上2点クリックで41点サンプリング、Chart.jsで描画
4. **ハザードマップ重ね合わせ** – 洪水浸水想定、浸水継続時間、土砂災害、高潮、津波（5種類）

### POI カテゴリー別ON/OFF表示
- 区役所／市役所（デフォルトON）
- 区の施設／市の施設
- 駅
- 公園
- 寺社仏閣
- 指定避難所（宿泊可）
- 指定緊急避難場所
- 広域避難場所
- 福祉避難所

### UX機能
- スマホ対応（ボトムシートUI、タッチ最適化、safe-area対応）
- 右上に緯度・経度・標高リアルタイム表示
- 標高スケール凡例
- 高さ強調スライダー（最大15倍）、ピッチ（角度）スライダー
- モバイル時は中心標高表示モード搭載

## 🛠️ 使用技術・データ

### フロントエンドライブラリ（CDN経由）
- **Leaflet 1.9.4** – 2Dマップ
- **MapLibre GL JS 4.7** – 3D地形表示
- **Chart.js 4.x** – 断面図描画
- **Font Awesome 6.4** – アイコン
- **Google Fonts** – Noto Sans JP / M PLUS Rounded 1c

### 使用データ（全て国土地理院・国交省のオープンデータ）
- GSI タイル：淡色地図、色別標高図、陰影起伏図、DEM PNG
- GSI 標高API：`https://cyberjapandata.gsi.go.jp/general/dem/scripts/getelevation.php`
- ハザードマップポータル：洪水L2、浸水継続時間、土砂災害、高潮、津波

### 技術ポイント
- **GSI DEM → terrain-rgb 変換**：`maplibregl.addProtocol('gsidem://', ...)` で実装、Canvas経由でRGB値を変換
- **認証不要・CORS対応**のオープンタイルのみ使用 → GitHub Pagesで完全動作
- **エラーハンドリング**：海上・圏外タイルの404を無視し空タイルを返却

## 📊 データモデル

このプロジェクトは**静的データのみ**使用（RESTful Table APIは未使用）。POI・避難所データは各HTMLファイル内のJavaScript配列として内包。

```javascript

  lat: 35.6693,
  lng: 139.4778,

}
```

## 🔌 オフライン対応方針

### 運用モード2種類

| モード | 要件 | 配布方法 |
|---|---|---|
| **通常版**（オンライン必須） | インターネット接続必要 | GitHub Pages閲覧 or ブラウザでHTMLを保存 |
| **オフライン版** | 完全オフライン動作 | 別ページ（`/offline/`）で詳細説明＋ダウンロード提供 |

### オフライン版の実装方式：**PWA（方式C）**

- **Service Worker** でライブラリ・HTMLをキャッシュ
- 一度オンラインで開いたエリアは以降オフライン閲覧可
- 国土地理院タイルのライセンス問題を回避（個人キャッシュ扱い）
- ファイルサイズ肥大化なし
- iOS/Androidで「ホーム画面に追加」→ アプリ化可能

### 方式選定の経緯（却下案含む）
- ❌ 方式A（タイル同梱なし）：地図が見えず実用性が低い
- ❌ 方式B（タイル同梱あり）：国土地理院の再配布許諾申請が必要・ファイル肥大化
- ✅ **方式C（PWA）**：ライセンス問題なし・軽量・現実的

## 🆕 フェーズ3 完了：共通コア化の仕組み

### 新しい自治体の追加フロー

**従来（単一HTML方式）**：
- 約55KBのHTMLを丸ごと新規作成（コード重複）
- バグ修正は4ファイル全て個別対応

**新方式（共通コア+JSON）**：
- `data/<slug>.json` を1枚作るだけ（約5KB）
- 共通ロジックは `/app/map.html` に集約（1ファイル更新で全自治体反映）
- URL: `/app/map.html?city=<slug>`

### JSON 追加例（新自治体対応の最短ルート）

```json
{
  "slug": "shinjuku",
  "name": "新宿区",
  "center": { "lat": 35.6939, "lng": 139.7036 },
  "theme": { "main": "#e91e63", "sub": "#ffc107" },
  "poi": {
    "yakusho": [ {"name": "新宿区役所", "lat": 35.6939, "lng": 139.7036} ],
    ...
  }
}
```

詳細は `/data/README.md` を参照。

### 進捗管理（`tokyo-index.json`）

各自治体の `status` フィールドで管理：
- `done`：既存単一HTML版で完成（府中・板橋・港・江東）
- `core`：共通コア方式で完成（千代田）
- `pending`：未作成

ポータルは自動的に読込み、完成済み自治体のみクリック可能に色付け表示。

## 🔌 PWA仕様（フェーズ2 完了）

### Service Worker キャッシュ戦略

| リソース種別 | 戦略 | キャッシュ名 |
|---|---|---|
| アプリシェル（HTML等） | Cache First | `shell-v1.0.0-20260423` |
| CDNライブラリ（jsDelivr等） | Cache First | `lib-v1.0.0-20260423` |
| 地図タイル（GSI等） | Stale While Revalidate | `tile-v1.0.0-20260423` |
| その他 | Network First | `shell-...` |

- バージョン番号 `CACHE_VERSION` で一括更新管理
- オフライン時、画像は透明1x1 PNG、その他は503 JSONを返却
